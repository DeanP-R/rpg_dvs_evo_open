◆ img_align_est_illumination_gain

bool svo::BaseOptions::img_align_est_illumination_gain = false

Estimate an affine transformation for illumination/exposure change. If you observe bad tracking because of illumination/exposure change, enabling these parameters might help. Normally it is OK to leave them as default.

◆ img_align_max_level

size_t svo::BaseOptions::img_align_max_level = 4

(!) During sparse image alignment (see [1]), we find the pose relative to the last frame by minimizing the photometric error between the frames. This KLT-style optimizer is pyramidal to allow more motion between two frames. Depending on the image size, you should increase this value. For images of the size 640x480 we set this value to 4. If your image is double the resolution, increase to 5, and so on.

◆ img_align_max_num_features

size_t svo::BaseOptions::img_align_max_num_features = 0

If you choose to extract many features then sparse image alignment may become too slow. You can limit the number of features for this step with this parameter to randomly sample N features that are used for alignment.

◆ img_align_min_level

size_t svo::BaseOptions::img_align_min_level = 2

(!) During sparse image alignment, we stop optimizing at this level for efficiency reasons. If you have the time, you can go down to the zero'th level. Depending on the image size, you should increase this value. For images of the size 640x480 we set this value to 2. If your image is double the resolution, increase to 3, and so on.

◆ img_align_prior_lambda_rot

double svo::BaseOptions::img_align_prior_lambda_rot = 0.0

If you are using a gyroscope and provide an attitude estimate together with the image in the function addImage() then this parameter specifies how much you trust your gyroscope. If you set it to 2.0 (default) it means that the gyroscope attitude is valued two times more than the actualy orientation estimate from the visual measurements.

◆ img_align_prior_lambda_trans

double svo::BaseOptions::img_align_prior_lambda_trans = 0.0

Internally, we have a very basic constant velocity motion model. similarly to lambda_rot, this parameter trades-off the visual measurements with the constant velocity prior. By default this weight is way below 1.

◆ img_align_use_distortion_jacobian

bool svo::BaseOptions::img_align_use_distortion_jacobian = false

Whether or not to include the distortion when calculating the jacobian. For small FoV pinhole projection, it is safe to leave it as false. For fisheye lens, set this to true.

◆ init_map_scale

double svo::BaseOptions::init_map_scale = 1.0

By default, when the VO initializes, it sets the average depth to this value. This is because from monocular views, we can't estimate the scale. If you initialize SVO with a downward-looking camera at 1.5m height over a flat surface, then set this value to 1.5 and the resulting map will be approximately in the right scale.

◆ init_use_att_and_depth

bool svo::BaseOptions::init_use_att_and_depth = false

By default, the orientation of the first camera-frame is set to be the identity (camera facing in z-direction, camera-right is in x-direction, camera-down is in y-direction) and the map scale is initialized with the option init_map_scale. However, if you would like to provide another initial orientation and inital scene depth, then activate this flag. If activated, you need to provide attitude and depth measurements using the functions addAttitudeMeasurement(), addDepthMeasurement() in the base class. This option is useful for in-flight initialization of SVO.

◆ max_n_kfs

size_t svo::BaseOptions::max_n_kfs = 10

The VO only keeps a list of N keyframes in the map. If a new keyframe is selected, then one furthest away from the current position is removed Default is 10. Set to 0 if all keyframes should remain in the map. More keyframes may reduce drift but since no loop-closures are actively detected and closed it is not beneficial to accumulate all keyframes.

◆ poseoptim_prior_lambda

double svo::BaseOptions::poseoptim_prior_lambda = 0.0

This is the same parameter as img_align_prior_lambda_rot but for the pose optimization instead of the sparse image alignment. Only used if you provide gyroscope measurements.

◆ poseoptim_thresh

double svo::BaseOptions::poseoptim_thresh = 2.0

(!) This parameter is the reprojection error threshold during pose optimization. If the distance between a feature and the projected pixel position of the corresponding 3D point is further than this threshold appart (on the zero'th level pyramid), then the feature is removed from the frame. With a good camera and image resolution of 640x480 a threshold of 2.0 is typically ok. If you use shitty cameras (rolling shutter), higher resolution cameras, cameras with imperfect calibration etc. you might increase this threshold. But first, check the tracefile for the average reprojection threshold. We made the experice that with GoPro cameras, we had to increase this threshold.

◆ poseoptim_using_unit_sphere

double svo::BaseOptions::poseoptim_using_unit_sphere = false

This parameter controls whether the pose optimizer works on:

  • unit plane: preferable for pinhole model
  • unit sphere: omnidirectional camera model(e.g. fisheye, catadioptric)

◆ quality_max_fts_drop

int svo::BaseOptions::quality_max_fts_drop = 40

If from one frame to the other we suddenly track much less features, this can be an indication that something is wrong and we set the stage to STAGE_RELOCALIZING and the tracking quality to TRACKING_INSUFFICIENT.

◆ quality_min_fts

size_t svo::BaseOptions::quality_min_fts = 50

Minimum number of features that should be tracked. If the number falls bellow then the stage is set to STAGE_RELOCALIZING and the tracking quality to TRACKING_INSUFFICIENT until we find more features again.

◆ relocalization_max_trials

size_t svo::BaseOptions::relocalization_max_trials = 100

Once we are in relocalization mode, we allow a fixed number of images to try and relocalize before we reset() and set to STAGE_PAUSED.

◆ structure_optimization_max_pts

int svo::BaseOptions::structure_optimization_max_pts = 20

By default SVO does not do bundle adjustment but it optimizes the pose and the structure separately. This is not optimal but much faster. This parameters specifies how many 3D points should be randomly selected at every frame and be optimized. For speed reasons, we don't optimize all points at every iteration. Set to -1 if you want to do so anyway.
